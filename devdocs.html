Architecture

System Overview

Mission Control runs on 208.87.135.84 (Ubuntu). Two main stacks:

Audit & Compliance Stack

  • mission-control-ui (Flask, port 8095) — Auth proxy, session management, serves HTML pages
  • device-audit-api (FastAPI, port 8096) — Audit CRUD, classification, PDF reports, governance API
  • DB: Supabase (PostgreSQL) — sites, devices, device_audits, audit_entries, remediation_actions, incidents

Network Monitoring Stack

  • backend_api (FastAPI, port 8000) — Probe agent API, device management, alerts, dashboard endpoints
  • mission_probe (Python agent) — Runs on Windows/Linux NUCs at client sites, scans subnets
  • DB: Supabase (PostgreSQL) — mc_sites, mc_network_devices, mc_alerts, mc_probe_agents, mc_device_observations
API

Key Endpoints

Monitoring API (port 8000)

  • GET /api/v1/dashboard/summary?site_code=X — Per-site dashboard data
  • GET /api/v1/dashboard/all-sites — Multi-site overview
  • GET /api/v1/dashboard/device/{id} — Device detail + observations + alerts
  • GET /api/v1/admin/devices?site_code=X — Device classification admin
  • GET /api/v1/dashboard/agents — All probe agents
  • POST /api/v1/devices/report — Batch device report from agent
  • POST /api/v1/heartbeat — Agent heartbeat

Project Hub (port 3010)

  • GET /api/projects — List all projects with health status
  • POST /api/projects — Create project
  • POST /api/projects/{id}/scaffold-tracker — Deploy tracker to local/remote server
  • GET /api/events/stream — SSE live updates
Deploy

Service Management

sudo systemctl restart mission-control-ui   # Auth proxy (port 8095)
sudo systemctl restart device-audit-api     # Audit API (port 8096)
sudo systemctl restart backend-api          # Monitoring API (port 8000)

Probe Agent Install (Windows)

.\install.ps1 -ApiKey <key> -SiteId <id> -SiteName "Name" -AgentId <agent> -Subnet "192.168.1.0/24"
