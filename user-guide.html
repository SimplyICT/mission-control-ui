Mission Control Platform

Mission Control is a centralised device audit and compliance management platform for early childhood education services. It manages device audits, compliance reporting, remediation tracking and incident management to support ACECQA Assessment & Rating obligations.

This guide covers every module available on the platform. Use the sidebar to jump to the section you need. This is a living document — it will be updated as features are added or workflows change.

Audit Running a Device Audit

A device audit records the security, photo retention and hardware condition of every device at a site. Audits produce a compliance score and generate remediation actions automatically.

Starting an audit

  1. Go to Audit Centre from Platform Home.
  2. Select the site from the dropdown.
  3. Click New Audit Batch. A new batch ID is created.
  4. Click Start Audit to open the Device Audit Form pre-filtered to that batch.

Filling in the Device Audit Form

  1. Select the site and the active batch at the top of the form.
  2. For each device, enter the serial number or select it from the device register.
  3. Complete all fields: device present, photos count, photo date, password/PIN status, camera sync, OneDrive sync, update status, hardware condition.
  4. Mark Ignore if the device is retired or excluded from compliance scoring.
  5. Click Save Entry for each device. The entry is saved to the database immediately.
  6. Repeat for all devices at the site.
Photo date should be the date photos were last reviewed/culled — not the date of the earliest photo. This is what drives the retention breach calculation.
Each site may have a custom photo retention threshold configured in Site Onboarding. The default is 31 days for photo age and 1,000 photos count. Check the site's policy settings before auditing.

Reviewing audit results

  1. Return to Audit Centre and select the batch.
  2. Click View Report to see a live summary of critical/warning/ok counts.
  3. Use the filter controls to focus on critical or warning items.

Full audit lifecycle: Audit → Finalize → Remediate → Report

The platform supports a complete audit workflow that captures findings, tracks remediation, and generates evidence-ready reports:

  1. Audit — Record device data and issue flags (password issues, photo counts, security concerns, missing devices) using the Device Audit Form for each device at the site.
  2. Finalize — On the Audit Reports page, click Finalize for the batch. This locks the current state as the audit snapshot and automatically generates remediation actions for every finding (missing devices, credential issues, photo retention breaches, hardware problems).
  3. Remediate — Click the Remediate button (purple) on the Audit Reports page to open the Remediation Management page. Here you can review all open findings, update their status (Open → In Progress → Resolved), add resolution notes, and link to each device's edit page to fix the underlying issue.
  4. Report — Click View on the Audit Reports page. The report now includes a Findings & Remediation section that shows each finding alongside its remediation status — what was found, whether it's been resolved, and what notes were recorded. This provides a clear before-and-after view for compliance evidence.
Remediation actions are generated once when the audit is finalized. If you reopen a finalized audit, make changes, and finalize again, new actions are created only for findings that don't already have an action. Duplicate (device + finding type) combinations are skipped automatically.

Reports Generating PDF Reports

PDF reports are executive-grade compliance documents suitable for ACECQA evidence packages and internal records. They include site-specific thresholds and, if policy data is configured, a Policy Compliance Checklist page.

Generate a PDF

  1. Go to Audit Reports from Platform Home.
  2. Select the site and locate the batch you want to report on.
  3. Click Generate PDF. The report is built server-side and downloaded automatically.
  4. Alternatively, open Director View and click the PDF button in the site row.

Devices CSV Import / Export

The Device Records page supports bulk export and import of all device records via CSV. This is useful for initial data migration, bulk updates, and maintaining records in spreadsheet tools.

Exporting device records

  1. Go to Device Records from Platform Home.
  2. Click Export CSV in the toolbar. A download begins immediately.
  3. The file contains all 26 device fields: device ID, site, asset tag, type, brand/model, name, purchase date, warranty expiry, assigned room and user, status, connectivity, IP/MAC addresses, storage, OS, and more.

Importing device records

  1. Prepare a CSV using the exported file as a template — the column headers must match exactly.
  2. Click Import CSV in the Device Records toolbar and select your file.
  3. The import uses upsert logic: rows with an existing device_id are updated; rows without a device_id are created as new records.
  4. A confirmation message shows the count of created and updated records.
To bulk-update a field (e.g. assigned room after a room reshuffle), export the CSV, update the column in your spreadsheet app, and re-import. Only changed fields are updated.
Do not modify the device_id column for existing records — this is the unique key used for upsert. Changing a device_id will create a duplicate record instead of updating the existing one.
ColumnNotes

site_id / site_nameEither field can be used to assign a site.
is_activeTRUE / FALSE. Inactive devices are hidden from audit dropdowns.

Compliance Remediation Tracker New

The Remediation Tracker converts audit findings into tracked action items with owners, priorities, due dates and resolution status. It is the formal record that corrective actions were taken — critical for ACECQA evidence.

Auto-generating actions from an audit

Remediation actions are now generated automatically when a batch is finalized — no separate step is needed. Every finding (missing device, credential issue, photo retention breach, hardware problem) gets a tracked action item. Duplicate finding types for the same device are skipped.

To manually generate actions for an already-finalized batch, navigate to the Remediation Management Page.

Creating a manual action

  1. Click New Action.
  2. Select the site, finding type, priority and due date.
  3. Enter a description and optionally assign to a staff member.
  4. Click Save.

Updating action status

  1. Find the action in the table (filter by site, status or priority as needed).
  2. Click the status dropdown in the action row and select the new status.
  3. For resolved actions, add resolution notes in the notes field before saving.
StatusMeaning
OpenAction identified, not yet started
In ProgressWork underway
ResolvedFinding corrected — add resolution notes
Accepted RiskRisk formally accepted by management — document rationale in notes
Overdue actions (past due date and not resolved) are highlighted in red on the Director View. Regularly review and close resolved items to keep the governance dashboard accurate.

Audit Audit Remediation Management New

The Audit Remediation page provides a focused view of all remediation actions for a specific audit batch. Access it by clicking the Remediate (purple) button next to any batch on the Audit Reports page.

Page layout

  1. Summary cards at the top show total actions, open/in-progress count and resolved count.
  2. Open & In Progress section lists all unresolved findings with status dropdowns and resolution note fields. Update status and add notes inline, then click Update to save.
  3. Resolved & Closed section lists completed actions with their resolution date and notes for reference.
  4. Each finding card shows the finding type (colour-coded: red for critical, amber for photo/hardware), description, device name (linked to the device edit page), serial number, room and device type.

Updating a finding from this page

  1. In the Open section, use the dropdown to change the action status: Open → In Progress → Resolved or Accepted Risk.
  2. Add resolution notes describing what was done to fix the issue.
  3. Click Update to save changes. The page refreshes and the action moves to the appropriate section.
  4. To fix the underlying device issue, click the device name link — it opens the Device Update page in a new tab where you can edit device details.

Viewing remediation in the Audit Report

After remediation actions are resolved, open the Audit Report (click View on the Audit Reports page). The report now includes a Findings & Remediation section that shows:

  • Each finding with its type and description
  • Current remediation status (Open / In Progress / Resolved)
  • Resolution notes and date if resolved
  • Summary totals: total actions, open count, resolved count
  • A link to the Remediation Management page for further action

This creates a complete compliance record: what was found during the audit, what actions were taken, and what the current state is.

Use the Remediation page as a workflow hub during the period between finalizing an audit and generating the final report. Work through each finding, update device records as fixes are applied, and mark actions resolved with detailed notes for the audit trail.
The device name in each finding card is a clickable link that opens the Device Update form (/device-update.html) in a new tab. This lets you fix device details (name, type, room assignment, status) without leaving the remediation workflow.

Compliance Incident Register New

The Incident Register is the formal log of privacy, data and device security incidents at each site. Incidents marked as Mandatory Reportable are flagged prominently and tracked through to regulatory notification.

Logging a new incident

  1. Go to Incident Register from the Compliance section.
  2. Click New Incident.
  3. Select the site and set the incident date.
  4. Choose the Incident Type (e.g. Privacy Breach, Missing Device with Data, Photo Policy Violation).
  5. Set Severity (Critical / High / Medium / Low).
  6. Write a description of what occurred.
  7. Toggle Children Affected if child data or images may be involved. Enter the affected count if known.
  8. Toggle Mandatory Reportable if the incident meets the threshold for regulatory notification.
  9. Enter response actions taken so far.
  10. Click Save Incident.

Mandatory reporting workflow

  1. When an incident is flagged as Mandatory Reportable, the register highlights it in red.
  2. Once reported to the regulator, update the Reported To field (e.g. "Queensland Department of Education") and set the Reported Date.
  3. Change status to Reported.
  4. Once investigated and closed, set status to Resolved and add resolution notes.
In Queensland, mandatory notifications under the Education and Care Services National Law must be made to the regulatory authority as soon as practicable. The Incident Register records the notification but does not replace direct contact with the regulator.

Linking an incident to an audit

If the incident arose from an audit finding (e.g. a photo retention breach that constitutes a privacy incident), you can link the incident to the audit batch and device serial in the incident form. This creates a traceable record from finding to incident to resolution.

Compliance Site Policy Configuration New

Each site can have its own photo retention thresholds and a policy documentation checklist. These values drive audit classification and appear in PDF reports.

Setting site thresholds

  1. Go to Site Onboarding from the Administration section.
  2. Select the site from the dropdown.
  3. Scroll to the Compliance Policy Configuration card.
  4. Set Photo Retention Days — devices with photos older than this are flagged Critical. Default: 31.
  5. Set Photo Count Threshold — devices with more photos than this receive a Warning. Default: 1,000.
  6. Set Audit Frequency (Days) — the expected interval between audits. Default: 90.
  7. Click Save All Changes.
If your service has a specific policy (e.g. 28-day photo retention), set this here. All subsequent audits and PDF reports for that site will use the site-specific threshold, not the system default.

Completing the policy checklist

  1. In the same Compliance Policy card, scroll to the Policy Checklist section.
  2. Toggle each item to green when the corresponding policy or document is in place and current.
  3. Click Save All Changes.
  4. The checklist status flows through to PDF reports (Policy Compliance Checklist page) and the Director View governance tiles.
Policy ItemWhat to check

Photo policy documentedA written photo and video policy exists and is accessible to staff
Staff privacy training currentAll staff have completed privacy/data handling training in the last 12 months

Executive Director View

The Director View provides a single-screen compliance overview for all sites. It is designed for directors, coordinators and executives who need a high-level picture without navigating individual audit reports.

Reading the dashboard

  1. Go to Director View from the Executive Dashboards section.
  2. The top row of tiles summarises all audits in the last 30 days: average score, risk counts, device and issue totals.
  3. The Latest Audit by Site table shows the most recent audit for each site — risk label, score, status, photo breaches, and a link to view or download the PDF.
  4. The Recent Audits table lists every audit batch in the last 30 days.
  5. The Compliance Overview tiles show open/overdue remediation actions, open/mandatory incidents, and policy gap count.
  6. The NQS Quality Area Mapping table maps all findings to relevant NQS areas — see below.

Admin Site Onboarding

Site Onboarding is where you configure all site-level details used across the platform.

Creating a new site

  1. Go to Site Onboarding.
  2. Click + Add New Site.
  3. Enter the site name and company name.
  4. Click Create Site.
  5. The site is now available in all dropdowns across the platform.

Updating site details

  1. Select the site from the dropdown.
  2. Update the Site Information card: name, location, address, contact, active status.
  3. Update SharePoint Integration if applicable.
  4. Update Compliance Policy Configuration (thresholds and policy checklist).
  5. Click Save All Changes — all cards are saved in a single action.

Admin SharePoint Integration

PDF audit reports can be automatically uploaded to a site's SharePoint document library after generation.

Setup

  1. In Azure AD, create an App Registration with Sites.ReadWrite.All application permission.
  2. In Site Onboarding, enable SharePoint Upload for the site.
  3. Enter the Tenant ID, Client ID, Client Secret, SharePoint Site URL and target folder.
  4. Click Auto-Discover to find the Drive ID automatically, or enter it manually.
  5. Click Test Connection to validate all steps before saving.
  6. Click Save All Changes.
Once configured, use the Upload to SharePoint button in Audit Reports to push a generated PDF directly to the site's library.

Security SOC Autopilot

The SOC Autopilot is an AI-powered security case management system that automatically triages alerts and proposes response actions.

Case List

  • KPI tiles: Last 24h cases, Critical count, High count, Awaiting Approval, Resolved, Average Triage Time.
  • Filter tabs: All, Pending (awaiting_approval), Open, Resolved.

Case Detail

Click View on any case to see full details:

  1. Header — Case ID, severity badge, status badge, alert count, creation date, AI confidence score, and MITRE technique mapping.

  3. Entities — Extracted indicators (IPs, hostnames, users) with type and role.
  4. Response Plan — AI-proposed actions with risk level, action type, target, and rationale.
  5. Executed Actions — Results of actions that have been run (status: Done, Failed, or Requested).
  6. Timeline — Chronological event log of case activity (triage, investigation, approval, rejection).

Changelog

Update this section each time features are added or changed.

June 2026 — v1.1

  • New Findings & Remediation Report Section — Audit report now shows a before-and-after table of findings with remediation status, resolution date and notes for each device
  • New Audit Remediation Management Page — Inline workflow to update action status, add resolution notes, and link to device editing, accessible via the Remediate button on Audit Reports
  • Updated Finalize triggers auto-remediation — Finalizing an audit now automatically generates remediation actions for all findings (missing devices, credential issues, photo breaches, hardware problems)
  • Updated Edit button on Audit Reports — Opens the Device Audit Form pre-loaded with a batch for continued data entry
  • Updated Device Audit Form — Device names are now clickable links to the Device Update page for editing device details in a new tab
  • New Wazuh SOC Dashboard — Full single-page SIEM dashboard with 14 views: Command Center, Agents, SCA, FIM, Vulnerabilities, MITRE ATT&CK, Rules & Decoders, Events & Alerts, Topology, Threat Intel, SOC Autopilot, Manager Health, Groups, and Help
  • New SOC Autopilot — AI-powered case management with automated triage, response plans, and approve/reject/execute workflow
  • New Threat Intelligence — AlienVault OTX integration dashboard with IOC tracking
  • New Wazuh API Proxy — Server-side authenticated proxy to Wazuh SOC API with automatic token management
  • New Security Operations section added to this User Guide

June 2026 — v1.0

  • New Two-Factor Authentication (TOTP) — QR code setup, per-user activate/disable, admin reset, brute-force lockout protection
  • New Remediation Tracker — auto-generate and manage audit remediation actions with priority, due date and resolution tracking
  • New Incident Register — log and manage ACECQA mandatory incidents with severity, mandatory-reportable flagging and regulatory notification tracking
  • New Site Policy Configuration — per-site photo retention thresholds and 9-item ACECQA policy checklist in Site Onboarding
  • New Governance Overview — 11 KPI tiles for remediation priority/severity, incident severity, and policy completeness in Director View
  • New Per-Site Policy Compliance table — shows confirmed/9, gaps, and missing policy names per site in Director View
  • New Audit Recency table — last audit date and overdue flag per site in Director View
  • New NQS Quality Area Mapping — findings mapped to NQS 2.2, 3.2, 4.2, 7.3 in Director View
  • New CSV Import / Export — bulk export and upsert-import for device records (26-column CSV)
  • Updated PDF reports include site-specific thresholds on cover page and Policy Compliance Checklist section
  • Updated Audit classification uses per-site photo retention thresholds — no longer relies on system defaults
  • Updated Director dashboard response time reduced from 55s to 2.4s (23× faster)
