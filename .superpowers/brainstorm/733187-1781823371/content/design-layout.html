Section 5: Tracker Scaffold Generator

When creating a new tracker, the backend generates 4 files in the target folder:

  1. index.html — Clone of the tracker UI, pre-configured with API port
  2. server.py — Standalone tracker server (Python stdlib, no dependencies needed)
  3. data.json — Empty projects data file
  4. tracker.service — systemd service file for auto-start

For remote servers: files are generated locally but the user deploys via SCP/rsync. Future: SSH-based remote deployment.
