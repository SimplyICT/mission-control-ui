Section 2: How Remote Deployment Works

Deployment Flow
  1. Generate scaffold — create index.html, server.py, data.json, tracker.service in a temp folder
  2. SSH connect — use stored SSH key/password to connect to remote server
  3. Create remote folder — mkdir -p /home/aiagent/project/tracker
  4. SCP files — copy scaffold files to remote folder
  5. Install service — copy tracker.service to /etc/systemd/system/ on remote
  6. Start tracker — run systemctl enable --now tracker.service on remote
  7. Verify — ping the tracker URL and confirm 200 response
  8. Save project — store all metadata in projects.json with health=online

Section 3: Stored Credential Management

SSH Credential Storage

Credentials are stored per-server in a separate JSON file (not projects.json):

// server-credentials.json (chmod 600 — access controlled)
{
  "servers": [
    {
      "host": "208.87.135.183",
      "port": 22,
      "username": "aiagent",
      "auth_method": "key",
      "key_path": "/home/aiagent/.ssh/id_rsa",
      "last_used": "2026-06-18T..."
    }
  ]
}

⚠️ Passwords stored encrypted using a server-side secret. SSH keys are referenced by path only — the key file stays on disk.

Section 6: "Deploy Tracker" Button Actions

Existing projects without a tracker (or needing re-deploy) get a deploy flow:

  1. Click "Deploy Tracker" on a project card
  2. Modal shows server config (pre-filled from stored credentials)
  3. Click "Test SSH Connection" — verifies credentials work
  4. Click "Deploy" — scaffold + SSH copy + service start + verification
  5. Card updates with health dot and "Open Tracker" button
